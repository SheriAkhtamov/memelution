# Memelution UI Reference Set

UI refresh. They were copied into the repository on 2026-06-05 so the redesign
does not depend on files remaining in Downloads.

## Visual Contract

- Treat the references as a coherent design system, not eight unrelated mockups.
- Preserve the recognizable three-column desktop shell, stable left navigation,
  contextual right rail, strong page headings, restrained orange brand accent,
  thin neutral borders, subtle shadows, and soft empty-state illustrations.
- Match referenced pages as closely as practical while keeping real product
  behavior, semantic HTML, accessibility, and responsive layouts.
- UI controls and text must remain code-native. Do not ship screenshots as UI.
- For pages without a direct screenshot, extend the same visual language using
  product-design judgment and the existing information architecture.
- Verify desktop and mobile behavior in a real browser. Visual fidelity alone is
  not enough if a core workflow or interaction breaks.
